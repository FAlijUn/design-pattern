状态模式（State Pattern） 是一种行为型设计模式，它允许一个对象在其内部状态改变时改变其行为
换句话说，状态模式允许对象在不同的状态下表现出不同的行为，而无需通过大量的条件语句来实现
状态模式的关键是将对象的不同状态封装成独立的状态类，每个状态类负责处理该状态下的行为，并且通过上下文对象来切换不同的状态

状态模式的结构
1. context（上下文）：维护当前状态的对象，并提供方法来切换不同的状态
2. state（状态接口）：声明所有的具体状态类都必须实现的接口，定义每个状态下的行为
3. ConcreteState（具体状态）：实现State接口，实现 State 接口，定义某一特定状态下的行为

状态机器的应用场景
1. 有限状态机：当系统有多个明确的状态，并且每个状态下的行为是不同的，状态模式非常适用。通过状态模式可以避免在代码中使用复杂的 if 或 switch 语句来判断当前状态
  电梯系统：电梯有多个状态（如停止、运行中、开门、关门等），每个状态下的行为不同。使用状态模式可以避免在每个方法中添加大量的条件判断
  打印机状态：打印机可能有不同的状态，如“空闲”、“打印中”、“暂停”等。每个状态下有不同的行为，可以使用状态模式管理这些状态
  网络连接状态：网络连接（如 TCP 连接）可能有多个状态（如“未连接”、“连接中”、“已连接”等），每个状态下的行为不同，可以通过状态模式实现

2. 需要处理对象行为的多种状态
  如果一个对象在多个状态下需要执行不同的操作，且状态的数量较多，使用状态模式可以使代码更加清晰、易于扩展
  游戏中的角色状态：游戏中的角色可能有多个状态（如“站立”、“跑步”、“跳跃”）。每个状态下，角色的行为（如移动速度、动作等）会不同，使用状态模式可以简化代码
  用户认证流程：用户在认证系统中可能有多个状态（如“未登录”、“登录中”、“已登录”）。每个状态下，认证系统的行为会不同
